Create another Docker engine in 5 minute

5 Minutes Clouding
2023-07-18
docker, linux

This blog explain how Docker work under the hood and from that we create a new similar tool.

What we cover here?

  • Storage binding - mapping the storage from physical host to container
  • Network mapping - mapping the network from container to physical host
  • Container image - split container image to a lot of reusable layer

Storage binding

Docker don’t create thing out of thin air. Most of if magic is depends on Linux namespace. Mount namespace allows us to create a simulated, isolated storage space that other processes can use.

Here we create a new storage namespace so we can map it to our container later.

Network mapping

Similar with Storage binding, we create our network namespace with id “some-sql”

To allow the MySQL instances running in the network namespaces can be access from the host, we will do the network bridge here. However, for the sake of our sanity, we will simply accept it as it is.

Container image

This feature don’t involve to Linux namespace. For each line of Dockerfile, we hash the command string to create the unique layer ID. If the layer already exist, we using cache instead of execute the build command
